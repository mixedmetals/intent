# Button

Buttons trigger actions or navigate to different pages.

## Overview

The Button component is the most fundamental interactive element. It supports multiple visual styles, sizes, and states.

## Examples

### Importance Levels

Use `importance` to indicate the priority of the action:

<Stack direction="row" gap="normal" style={{ marginBottom: 24 }}>
  <Button importance="primary">Primary</Button>
  <Button importance="secondary">Secondary</Button>
  <Button importance="tertiary">Tertiary</Button>
</Stack>

```tsx
<Stack direction="row" gap="normal">
  <Button importance="primary">Primary</Button>
  <Button importance="secondary">Secondary</Button>
  <Button importance="tertiary">Tertiary</Button>
</Stack>
```

### Sizes

Three sizes to fit different contexts:

<Stack direction="row" gap="normal" align="center" style={{ marginBottom: 24 }}>
  <Button size="small">Small</Button>
  <Button size="medium">Medium</Button>
  <Button size="large">Large</Button>
</Stack>

```tsx
<Stack direction="row" gap="normal" align="center">
  <Button size="small">Small</Button>
  <Button size="medium">Medium</Button>
  <Button size="large">Large</Button>
```

### Full Width

Expand to fill the container:

<div style={{ width: 300, marginBottom: 24 }}>
  <Button importance="primary" fullWidth>Full Width Button</Button>
</div>

```tsx
<Button importance="primary" fullWidth>
  Full Width Button
</Button>
```

### Disabled State

<Stack direction="row" gap="normal" style={{ marginBottom: 24 }}>
  <Button disabled>Disabled</Button>
  <Button importance="primary" disabled>Disabled Primary</Button>
</Stack>

```tsx

```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `importance` | `'primary'` \| `'secondary'` \| `'tertiary'` | `'secondary'` | Visual priority |
| `size` | `'small'` \| `'medium'` \| `'large'` | `'medium'` | Button size |
| `fullWidth` | `boolean` | `false` | Expand to fill container |
| `disabled` | `boolean` | `false` | Disable interactions |
| `onClick` | `() => void` | - | Click handler |

## Usage Guidelines

### Choosing Importance

- **Primary** - The main action on a page or section
- **Secondary** - Alternative or supporting actions
- **Tertiary** - Low-priority actions, often paired with other buttons

### Common Patterns

**Dialog actions:**
```tsx
<Stack direction="row" gap="normal" justify="end">
  <Button importance="tertiary" onClick={onCancel}>
    Cancel
  </Button>
  <Button importance="primary" onClick={onConfirm}>
    Confirm
  </Button>
</Stack>
```

**Form submission:**
```tsx
<Button 
  importance="primary" 
  fullWidth 
  disabled={isSubmitting}
>
  {isSubmitting ? <Spinner size="small" /> : 'Submit'}
</Button>
```

## Accessibility

- Uses native `<button>` element
- Respects keyboard navigation
- Includes focus indicators
- Supports `disabled` attribute

# Dialog

Pre-configured modal for common confirmation patterns.

## Overview

Dialog is a specialized Modal component with pre-built patterns for:
- Alert dialogs (information, warnings, errors)
- Confirmation dialogs (yes/no, ok/cancel)
- Form dialogs with standard actions

Think of Dialog as Modal with opinions, while Modal gives you full control.

## Alert Dialog

Simple notification with single action:

```tsx
import { useState } from 'react'
import { Button, Dialog } from 'intent-react'

function AlertExample() {
  const [open, setOpen] = useState(false)
  
  return (
    <>
      <Button onClick={() => setOpen(true)}>Show Alert</Button>
      
      <Dialog.Alert
        open={open}
        onOpenChange={setOpen}
        title="Subscription Expired"
        description="Your subscription has expired. Renew to continue accessing premium features."
        action={{ label: 'Renew Now', onClick: handleRenew }}
      />
    </>
  )
}
```

## Confirm Dialog

Two-action confirmation:

```tsx
<Dialog.Confirm
  open={open}
  onOpenChange={setOpen}
  title="Delete Account"
  description="Are you sure you want to delete your account? This action cannot be undone."
  confirm={{ 
    label: 'Delete', 
    onClick: handleDelete,
    danger: true 
  }}
  cancel={{ label: 'Keep Account' }}
/>
```

## Form Dialog

Dialog with form fields:

```tsx
<Dialog.Form
  open={open}
  onOpenChange={setOpen}
  title="Create Project"
  description="Enter a name for your new project."
  onSubmit={handleSubmit}
>
  <Field label="Project Name">
    <Input placeholder="My Project" />
  </Field>
  <Field label="Description">
    <Textarea placeholder="Optional description" />
  </Field>
</Dialog.Form>
```

## Dialog Variants

### Info

```tsx
<Dialog.Alert
  variant="info"
  title="New Feature Available"
  description="Check out the new dashboard layout!"
  action={{ label: 'Take Tour' }}
/>
```

### Success

```tsx
<Dialog.Alert
  variant="success"
  title="Payment Successful"
  description="Your payment has been processed."
  action={{ label: 'View Receipt' }}
/>
```

### Warning

```tsx
<Dialog.Confirm
  variant="warning"
  title="Unsaved Changes"
  description="You have unsaved changes. Are you sure you want to leave?"
  confirm={{ label: 'Leave', danger: true }}
  cancel={{ label: 'Stay' }}
/>
```

### Danger

```tsx
<Dialog.Confirm
  variant="danger"
  title="Delete Repository"
  description="This will permanently delete the repository and all associated data."
  confirm={{ label: 'Delete', danger: true }}
  cancel={{ label: 'Cancel' }}
/>
```

## Props

### Dialog.Alert

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | State change callback |
| `variant` | `'info'` \| `'success'` \| `'warning'` \| `'danger'` | `'info'` | Dialog style |
| `title` | `string` | required | Dialog title |
| `description` | `string` | - | Dialog description |
| `action` | `{ label: string, onClick?: () => void }` | required | Primary action |

### Dialog.Confirm

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | State change callback |
| `variant` | `'default'` \| `'danger'` | `'default'` | Dialog style |
| `title` | `string` | required | Dialog title |
| `description` | `string` | - | Dialog description |
| `confirm` | `{ label: string, onClick?: () => void, danger?: boolean }` | required | Confirm action |
| `cancel` | `{ label: string, onClick?: () => void }` | required | Cancel action |

### Dialog.Form

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | State change callback |
| `title` | `string` | required | Dialog title |
| `description` | `string` | - | Dialog description |
| `onSubmit` | `(e: FormEvent) => void` | required | Form submit handler |
| `submitLabel` | `string` | `'Save'` | Submit button text |
| `children` | `ReactNode` | - | Form content |

## Dialog vs Modal

Use **Dialog** when:
- You need standard confirmation patterns
- Quick implementation with consistent styling
- Simple alert/confirm/form use cases

Use **Modal** when:
- You need full control over content
- Custom layouts or complex interactions
- Non-standard dialog patterns

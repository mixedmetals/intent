# Portal

Render content outside the component tree.

## Overview

Portals provide a way to render children into a DOM node that exists outside the hierarchy of the parent component. This is useful for modals, tooltips, and overlays.

## Examples

### Basic Portal

```tsx
import { Portal } from 'intent-react';

<Portal>
  <Modal />
</Portal>
```

### To Specific Container

```tsx
<Portal container={document.getElementById('modal-root')}>
  <Tooltip />
</Portal>
```

```tsx

  <div style={{ position: 'fixed', bottom: '20px', right: '20px' }}>
    Toast notification
  </div>
</Portal>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Content to portal |
| `container` | `Element` | `document.body` | Target container |

## Design Notes

- Portal doesn't generate CSS
- It moves DOM elements to the specified container
- Useful for escaping overflow:hidden parents
- Required for modals and overlays
- Preserves React event bubbling

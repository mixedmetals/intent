# Portal

Renders children into a DOM node outside the component hierarchy.

## Examples

### Basic Portal

```tsx
import { Portal } from 'intent-react'

function Modal({ children, open }) {
  if (!open) return null
  
  return (
    <Portal>
      <div className="modal-overlay">
        <div className="modal-content">
          {children}
        </div>
      </div>
    </Portal>
  )
}
```

### Custom Container

```tsx
// First, create a container
<div id="modal-root" />

// Then portal to it
<Portal containerId="modal-root">
  <ModalContent />
</Portal>
```

### Tooltip Portal

```tsx
function Tooltip({ children, content }) {
  const [show, setShow] = useState(false)
  
  return (
    <>
      <span 
        onMouseEnter={() => setShow(true)}
        onMouseLeave={() => setShow(false)}
      >
        {children}
      </span>
      {show && (
        <Portal>
          <div className="tooltip" style={{ position: 'fixed', top: y, left: x }}>
            {content}
          </div>
        </Portal>
      )}
    </>
  )
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | required | Content to portal |
| `containerId` | `string` | `'portal-root'` | Target container ID |

## Setup

Add a portal root to your HTML:

```html
<body>
  <div id="root"></div>
  <div id="portal-root"></div>
</body>
```

Or let Portal create it automatically.

## Why Use Portals?

- Escape CSS stacking contexts (z-index)
- Avoid overflow:hidden clipping
- Keep DOM semantic (modals at body level)
- Prevent event bubbling issues

## CSS Output

No specific CSS. Portal doesn't render any DOM elements itself.

```css
/* Portal content inherits styles from target container */
#portal-root {
  position: relative;
  z-index: 9999;
}
```

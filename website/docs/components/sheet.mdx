# Sheet

Bottom sheet for mobile-first designs.

## Overview

Sheet is a specialized overlay component that slides up from the bottom of the screen. It's the mobile equivalent of a Drawer and follows platform conventions for iOS/Android bottom sheets.

## Example

```tsx
import { useState } from 'react'
import { Button, Sheet, Text } from 'intent-react'

function Example() {
  const [open, setOpen] = useState(false)
  
  return (
    <>
      <Button onClick={() => setOpen(true)}>Open Sheet</Button>
      
      <Sheet
        open={open}
        onOpenChange={setOpen}
        title="Options"
      >
        <Text>Sheet content goes here.</Text>
      </Sheet>
    </>
  )
}
```

## With Handle

Sheets typically show a drag handle on mobile:

```tsx
<Sheet
  open={open}
  onOpenChange={setOpen}
  showHandle
  title="Filter"
>
  <Stack gap="normal">
    <Field label="Sort by">
      <Select options={sortOptions} />
    </Field>
    <Field label="Category">
      <Select options={categoryOptions} />
    </Field>
  </Stack>
</Sheet>
```

## Sizes

Sheets can be different heights:

| Size | Height | Use Case |
|------|--------|----------|
| `auto` | Fit content | Short options/lists |
| `sm` | 25% viewport | Quick actions |
| `md` | 50% viewport | Forms, filters |
| `lg` | 75% viewport | Detail views |
| `full` | 100% viewport | Full content |

```tsx
<Sheet size="md" open={open} onOpenChange={setOpen}>
  ...
</Sheet>
```

## Snap Points

Sheets can snap to specific heights:

```tsx
<Sheet
  open={open}
  onOpenChange={setOpen}
  snapPoints={['25%', '50%', '100%']}
  defaultSnapPoint={1}  // Start at 50%
>
  <Text>Swipe up to expand</Text>
</Sheet>
```

## Mobile Action Sheet

Common pattern for mobile action selection:

```tsx
<Sheet
  open={open}
  onOpenChange={setOpen}
  variant="actions"
  actions={[
    { label: 'Share', icon: 'share', onClick: handleShare },
    { label: 'Copy Link', icon: 'link', onClick: handleCopy },
    { label: 'Add to Favorites', icon: 'heart', onClick: handleFavorite },
    { label: 'Delete', icon: 'trash', danger: true, onClick: handleDelete },
  ]}
  cancel={{ label: 'Cancel' }}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | State change callback |
| `title` | `string` | - | Sheet title |
| `size` | `'auto'` \| `'sm'` \| `'md'` \| `'lg'` \| `'full'` | `'auto'` | Sheet height |
| `showHandle` | `boolean` | `true` | Show drag handle |
| `dismissible` | `boolean` | `true` | Allow swipe to dismiss |
| `snapPoints` | `string[]` | - | Snap point heights |
| `defaultSnapPoint` | `number` | `0` | Initial snap point index |
| `variant` | `'default'` \| `'actions'` | `'default'` | Sheet variant |
| `actions` | `Action[]` | - | Action items (for actions variant) |
| `cancel` | `{ label: string }` | - | Cancel button (for actions variant) |

### Action Type

```typescript
interface Action {
  label: string
  icon?: string
  onClick: () => void
  danger?: boolean
  disabled?: boolean
}
```

## Sheet vs Drawer

Use **Sheet** when:
- Targeting mobile-first design
- Following iOS/Android conventions
- Need snap points and gestures
- Action selection on mobile

Use **Drawer** when:
- Desktop or responsive layouts
- Side navigation or details panel
- Persistent content that slides

## Responsive Pattern

```tsx
function ResponsiveOverlay() {
  const [open, setOpen] = useState(false)
  const isMobile = useMediaQuery('(max-width: 768px)')
  
  return isMobile ? (
    <Sheet open={open} onOpenChange={setOpen}>
      <FilterContent />
    </Sheet>
  ) : (
    <Drawer open={open} onOpenChange={setOpen}>
      <FilterContent />
    </Drawer>
  )
}
```
